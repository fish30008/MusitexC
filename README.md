## Parser Class for Music Language

The `Parser` class is responsible for processing a sequence of tokens and converting them into an Abstract Syntax Tree (AST). This AST represents various components of a music-related language, including metadata, macros, variables, and tracks.

### Key Features

- **Initialization**: 
  - The parser is initialized with a list of tokens. 
  - It keeps track of the current token and the position in the token list.

- **Advancing Tokens**:
  - The `advance` method moves the parser to the next token, updating the current token.
  - The `peek` method allows checking the next token without advancing.
  - The `match` method checks if the current token matches a given type and advances the parser if it does.

- **Handling Newlines**:
  - The `skip_newlines` method allows the parser to skip over any newline tokens, which helps manage formatting in the input.

### Parsing Flow

The `parse` method is the core of the parsing process. It sequentially processes the tokens to construct different components of the program, handling:

- **Metadata**: Information like `title`, `tempo`, and `key`.
- **Macros**: Definitions of reusable sequences of music instructions.
- **Variables**: Assignments of values (such as notes) to variables.
- **Tracks**: Definitions of musical tracks, which consist of notes and other elements.

The `parse` method checks the type of each token, advancing or processing it based on its role in the language. It handles different scenarios such as parsing metadata, macro definitions, and track definitions.

### Parsing Specific Elements

- **Metadata Parsing**:
  - The parser extracts metadata properties like `title`, `tempo`, and `key` using methods like `parse_title`, `parse_tempo`, and `parse_key`.

- **Macro Parsing**:
  - Macros are defined with or without parameters, and the parser handles both cases. It parses the macro's name, parameters, and body (which can include notes or other macros).

- **Variable Parsing**:
  - Variables are assigned values, which are typically notes. The parser processes these assignments and associates them with variable names.

- **Track Parsing**:
  - Tracks consist of musical elements like notes, macros, and other structures. The parser processes these elements until the end of the track.

### Error Handling

- **Unexpected Tokens**: 
  - If the parser encounters an unexpected token, it raises a `SyntaxError`, providing information about the expected token and its location in the input.
  
- **Macro and Variable Syntax**:
  - The parser expects a specific format for macro calls and variable assignments. Errors are raised if the syntax deviates from the expected structure.

### Helper Methods

- **`parse_macro_body`**:
  - This method parses the content within a macro definition or a macro call, handling notes, variables, or other macro calls.

- **`parse_separator`**:
  - It handles the parsing of separator tokens (like dashes) used to separate different parts of the program.

### Overall Flow

The parser begins by processing metadata, then moves on to handle macros, variables, and tracks. Each component is parsed using specific methods tailored to its structure. After all tokens are processed, the parser returns a complete program represented by an AST that includes all metadata, macros, variables, and tracks.

### `advance(self)`

```python
def advance(self):
    self.pos += 1
    if self.pos < len(self.tokens):
        self.current_token = self.tokens[self.pos]
        return self.tokens[self.pos - 1]
    return self.tokens[-1]  # Return EOF token
```

* Advances the position by one token.
* If there are more tokens, updates the `current_token`.
* If at the end of the list, returns an EOF token.

### `peek(self, offset=1)`

```python
def peek(self, offset=1):
    pos = self.pos + offset
    if pos < len(self.tokens):
        return self.tokens[pos]
    return self.tokens[-1]  # Return EOF token
```

* Returns the token at a given `offset` from the current position, or the EOF token if the offset is beyond the list length.

### `match(self, token_type)`

```python
def match(self, token_type):
    if self.current_token.type == token_type:
        return self.advance()
    elif self.current_token.type != token_type:
        return False
    else:
        raise SyntaxError(
            f"Expected {token_type}, got {self.current_token.type} at line {self.current_token.line}, column {self.current_token.column}"
        )
```

* Matches the current token with the given `token_type`.
* Advances to the next token if matched.
* Raises a `SyntaxError` if the token type doesn't match.

def parse_title(self, metadata):
    """Parse the title property"""
    self.match(TokenType.KW_TITLE)
    self.match(TokenType.COLON)
    title_token = self.match(TokenType.STRING)
